📱 Context info

Protocol, AudioContext support, iOS detection

Test 1 — Oscillator (simplest possible)

Creates AudioContext + plays a 440hz beep inline in the tap handler. If this works, Web Audio works on this device.

Test 2 — Buffer noise click

Exactly what the app does: white noise burst through a bandpass filter. Tests whether the buffer approach works.

Test 3 — Reuse context (app pattern)

Creates context once on first tap, reuses on later taps (mirrors app's initAudioContext / resumeAudioContext pattern). Tests whether a reused context can still play.

Test 4 — Context created BEFORE gesture

Context created at page load (not in tap). Play attempted in tap handler. This should FAIL on iOS — confirms the restriction is real.

Test 5 — unlock on first tap, play on second

First tap creates + resumes context + plays a silent buffer (the "iOS unlock" pattern). Second tap plays the real sound. Tests whether the unlock trick works.

Test 6 — MediaStream routing (THE FIX)

Web Audio → audioCtx.destination goes to the RINGER channel (silenced by mute switch + ringer volume). This routes through MediaStreamAudioDestinationNode → <audio> element instead, which uses the MEDIA channel. Tap once to init, then tap again to play the click.
